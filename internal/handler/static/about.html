The Problem

AI agents forget everything between sessions. Existing memory solutions have fundamental problems:

Flat Text Files

Hermes MEMORY.md: 2200 chars. Claude's MEMORY.md: 10KB. You fill it up, then start deleting things you might need later. No search. No structure. No version history.

Vector-Only Databases

ChromaDB, Pinecone, Weaviate: great at semantic search, terrible at exact matches. Can't find "port 8081" reliably. No temporal history. No graph relationships between memories.

Cloud Memory APIs

Mem0, Zep, MemoryGPT: your project data on someone else's servers. API keys, rate limits, latency. Vendor lock-in. Your decisions about your infrastructure sitting on AWS.

The Solution

MindBank is a local-first graph memory system that combines three search methods, temporal versioning, and namespace isolation in a single Go binary.

How It Works

Hybrid Search (3 methods combined)

1. Full-Text Search — PostgreSQL tsvector with synonym expansion. "bugs" finds "bug", "issue", "problem". 1ms latency.

2. Vector Search — nomic-embed-text 768d embeddings via Ollama. "What stores data?" finds "ClickHouse" and "PostgreSQL".

3. RRF Fusion — Reciprocal Rank Fusion combines both with k=60. Best of exact matching AND semantic understanding.

Temporal Versioning

Every update creates a new version. Old versions are soft-deleted (valid_to set), never hard-deleted. Zero data loss.

Version chains via predecessor_id. Full audit trail. Purge old versions when you want.

// Update node → old version preserved
PUT /api/v1/nodes/abc123
→ Old: valid_to = now()
→ New: id=def456, version=2, predecessor=abc123
→ Edges: automatically relinked to def456

Key Advantages

1. 100% recall on known queries. FTS with synonym expansion finds "bugs" even when the node says "problem". Vector search finds semantically related memories even with different words. RRF fusion combines both.
2. Namespace isolation — zero cross-contamination. Working on Klixsor? You only see Klixsor memories. Switch to MindBank project? Only MindBank memories. Auto-detected from your working directory. No configuration needed.
3. Temporal versioning — zero data loss. Updated a node? Old version is preserved. Deleted accidentally? Soft-delete only. Full audit trail. Purge when you want.
4. Graph relationships — not flat lists. Memories connect to each other via semantic edges: decisions → decided_by → projects, problems → contradicts → decisions. 2191 edges across 200 nodes. Graph traversal finds related memories you didn't ask about.
5. Local-first — your data stays on your machine. Postgres in Docker, Ollama for embeddings, Go binary for API. No API keys, no cloud accounts, no vendor lock-in. Works offline.

Quick Start

# 1. One-command install
bash scripts/setup.sh

# 2. Start the API

# 3. Verify
curl http://localhost:8095/api/v1/health
# → {"status":"ok","postgres":"connected","ollama":"connected"}

# 4. Open dashboard
open http://localhost:8095

# 5. Start Hermes with memory
hermes chat
# → "## MindBank Memory (865 tokens)" injected automatically

What Gets Stored

Automatic (plugin handles)

Every conversation turn — smart labels extracted from user messages, type detection (decision/problem/advice)

Built-in memory writes — mirrors MEMORY.md/USER.md changes to the graph

Session extraction — scans for keywords like "decided", "deploy", "bug", "fix"

Unanswered questions — searches that return no results are stored for future reference

On-demand (tool calls)

mindbank_store — store any decision, fact, preference, or problem

mindbank_search — hybrid FTS + semantic search with namespace filter

mindbank_ask — natural language question with graph context

mindbank_snapshot — get pre-computed wake-up context

mindbank_neighbors — explore graph connections

Security

Authentication

Optional API key via MB_API_KEY env var. When set, all requests require Authorization: Bearer <key> header. Health endpoint always accessible.

Rate Limiting

100 requests per minute per IP. Prevents abuse. Returns 429 with Retry-After header when exceeded.

CORS

Whitelist only: localhost, 127.0.0.1, *.mindbank.local. No wildcard origins.

Input Validation

Label: 500 chars max. Content: 50KB max. Summary: 1KB max. Batch: 100 nodes/200 edges max. Prevents DoS.
